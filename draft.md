# 📄 프로젝트 문제정의 및 기획서

## 1. 제안 주제
**폐쇄망 환경을 위한 On-Device AI 기반 민원 분석 및 자동 답변 생성 시스템**

## 2. 프로젝트 추진 배경 및 필요성

### 2.1. 공공 데이터의 보안성 한계 및 폐쇄망 처리의 당위성
- 주요 국가 기관 및 지자체에 접수되는 민원 데이터에는 주민등록번호, 연락처, 개인 식별 정보 등 고도로 민감한 개인정보가 포함되어 있습니다.
- 이를 ChatGPT, Claude와 같은 외부 클라우드 기반 LLM API로 전송할 경우 심각한 보안 위반 및 데이터 유출 리스크가 발생합니다.
- 따라서 데이터 파기가 보장되고 외부 인터넷 연결 없이 오프라인(폐쇄망) 환경에서 독립적으로 구동되는 **On-Device(On-Premise) AI 시스템**이 필수적입니다.

### 2.2. 기존 키워드 기반 시스템의 한계 극복
- 기존의 단순 키워드 매칭이나 룰 규칙(Rule-based) 분류 시스템은 민원인이 작성한 복잡한 문맥이나 비정형적인 텍스트, 오탈자, 감정적 표현 등을 정확히 파악하는 데 한계가 있습니다.
- LLM의 자연어 이해(NLU) 능력을 활용하여 문맥 이면의 실제 민원 요지를 정확히 추출해야 합니다.

### 2.3. 행정 업무의 병목 현상 해소 및 효율성 극대화
- 담당 공무원이 수백, 수천 건의 과거 민원 사례와 법령을 일일이 검색하고 대조하여 답변을 작성하는 데 막대한 행정력이 낭비되고 있습니다.
- 의미 기반 검색(Semantic Search)을 통해 과거의 유사한 해결 사례와 관련 법령을 즉각적으로 참조하고, 이를 바탕으로 초안을 자동 생성하는 시스템이 도입되면 업무 처리 속도가 비약적으로 향상됩니다.

### 2.4. sLLM(소형 대규모 언어 모델)을 통한 경제적인 AI 도입
- 수백억 개의 파라미터를 갖는 초대형 클라우드 모델 대신, 수십억 개 수준의 파라미터를 갖는 오픈소스 **sLLM(Gemma 2, Llama 3 등)**을 특정 도메인(행정/민원)에 맞게 파인튜닝(Fine-tuning)함으로써, 고가의 서버 장비 없이도 유지비가 저렴하고 빠르며 정확한 서비스 구현이 가능합니다.

## 3. 세부 개발 요구사항

### 3.1. 도메인 특화 데이터 확보 및 파인튜닝 (SFT)
- **크롤링 및 데이터 셋 구축**: 각 지자체 웹사이트의 '전자민원창구', '국민신문고' 등에서 공개된 민원 글과 공식 답변 데이터를 수집하는 자동화 크롤러 개발.
- **데이터 전처리 및 비식별화**: 수집된 데이터 내에 혹여 존재할 수 있는 개인정보를 PII(Personally Identifiable Information) 마스킹 알고리즘을 통해 철저히 비식별화 처리 구축.
- **모델 지도 미세 조정 (Supervised Fine-Tuning)**: 수집된 질의-응답 데이터를 바탕으로 오픈소스 sLLM을 파인튜닝하여, 행정 문서 특유의 어투와 형식을 학습시킴 (LoRA/QLoRA 등 파라미터 효율적 미세조정 기법 적용).

### 3.2. RAG 기반 문맥 이해 및 표준 답변 생성 엔진 구현
- **의미 기반 벡터 검색 (Vector Search)**: FAISS를 활용하여 과거 유사 민원과 모범 답변 사례를 벡터 DB화하고, 새로운 민원 접수 시 코사인 유사도 등을 통해 가장 적합한 참고 자료 추출.
- **프롬프트 엔지니어링**: 공공기관 답변의 표준 서식(인사말, 본문, 관련 법령 안내, 맺음말 등)을 시스템 프롬프트로 템플릿화하여 일관된 톤앤매너(Tone & Manner) 유지.
- **초안 즉시 생성**: 사용자가 민원을 입력하면 검색된 유사 사례를 바탕으로 논리적이고 유려한 고품질의 답변 초안을 수 초 내에 자동 생성.

### 3.3. 고성능 On-Device 구동을 위한 추론 최적화 (Optimization)
- **모델 경량화 및 양자화 (Quantization)**: 메모리 사용량을 줄이고 추론 속도를 높이기 위해 4-bit 또는 8-bit 양자화 기법 적용.
- **가속 프레임워크 도입**: ONNX, TensorRT 및 Ollama 등 고성능 추론 엔진을 연동하여, 슈퍼컴퓨터가 아닌 일반적인 시스템 환경에서도 원활하게 작동하도록 서빙 환경 최적화.

### 3.4. 사용자 친화적 UI/UX 시스템 구축 (Frontend)
- **Streamlit 기반 웹 애플리케이션**: 복잡한 설치 없이 브라우저에서 바로 접근 가능한 대시보드 UI 제공.
- **주요 기능 인터페이스**: 
  - 텍스트 입력 및 파일 업로드 창
  - AI가 분석한 핵심 요약 및 유사 민원 사례 추천(Reference) 결과 창
  - 최종 생성된 답변 초안 제공 및 실시간 텍스트 에디터(수정 기능)
  - 클립보드 복사(Copy) 및 공유 기능
- **이식성을 위한 Docker 배포 패키징**: 모든 모듈(Frontend, 추론 엔진, Vector DB 등)을 Docker Compose로 컨테이너화하여, 타 기관의 폐쇄망 환경에서도 `docker-compose up -d` 하나의 명령어로 즉시 배포할 수 있는 이식성 확보.

## 4. 필요 핵심 기술 스택 (Tech Stack)

| 분류 | 도입 기술 | 활용 목적 |
|---|---|---|
| **AI & Modeling** | HuggingFace, Llama 3 / Gemma 2 | 오픈소스 sLLM 기반 생성 모델 구축 |
| **Fine-Tuning** | PEFT, LoRA, QLoRA | 효율적인 파라미터 미세 조정 |
| **Optimization** | Ollama, ONNX, TensorRT, Quantization | 로컬 환경에서의 모델 추론 속도 가속 및 경량화 |
| **Vector DB / RAG**| FAISS, LangChain | 의미 기반 유사도 검색 및 검색 증강 생성 파이프라인 |
| **Frontend & UI** | Streamlit | 파이썬 기반 데이터/AI 직관적 웹 화면 구축 |
| **Infra & DevOps** | Docker, Docker Compose | 환경 격리, 폐쇄망 통합 배포 및 컨테이너 가상화 |

## 5. 기대 효과 및 향후 발전 방향

- **업무 생산성 혁신**: 단순 반복적인 답변 작성 시간을 대폭 단축시켜, 공무원이 보다 복합적이고 중요한 대민 서비스에 집중할 수 있도록 지원.
- **응대 품질의 상향 평준화**: 신규 실무자도 AI 가이드를 통해 다년간 숙련된 공무원 수준의 정확하고 일관된 민원 답변을 작성 가능.
- **보안 및 규제 철저 준수**: 모든 데이터 분석 및 추론이 서버 내부(오프라인 망)에서 이루어지므로 개인정보보호법 및 망분리 규정 완벽 준수.
- **향후 확장성**: 텍스트 기반 민원에서 출발하여, 추후 음성 인식(STT) 기술을 결합한 '콜센터 녹취록 자동 분석 시스템' 등으로 손쉽게 확장 가능.
