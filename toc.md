# 제안서 (Proposal)

## ℹ️ 기본 정보 (Basic Info)
- **프로젝트명**: AI-Civil-Affairs-Systems (온디바이스 AI 데이터 심층 분석 및 검색 시스템)
- **개요**: 인터넷이 차단된 로컬 환경에서 공공 민원 데이터를 구조화, 검색, 질의응답까지 일관 처리하는 보안 특화 AI 조력자
- **동기**: 공공 민원의 특성상 외부 API 사용 불가로 인한 한계를 극복하고, 담당자의 반복적인 과거 유사 사례 탐색 및 분류 시간을 획기적으로 감축하기 위함
- **예상 결과물**: Streamlit UI, 구조화/검색/QA 파이프라인, 그리고 API가 통합된 실행 가능한 로컬 데모 앱

## 📋 주요 기능 (Key Features)
- **핵심 기능 1 (민원 텍스트 구조화 및 NER)**: 비정형 원문 데이터에서 4요소(Observation, Result, Request, Context) 및 중요 엔티티를 추출하여 일관된 스키마(JSON)로 자동 구조화
- **핵심 기능 2 (의미 기반 하이브리드 RAG 검색)**: 자연어 질의 기반 임베딩과 메타데이터 필터를 활용하여 목표치(Recall@5 0.85)를 만족하는 높은 품질의 유사 민원 검색
- **핵심 기능 3 (출처 하이라이팅 답변 생성)**: 검색된 청크(Chunk)를 기반으로 로컬 LLM이 환각 없이 요약/답변을 생성하고, 사용자가 신뢰할 수 있도록 답변에 출처와 원문 일부를 하이라이트 제공

## 📚 기술 스택 (Tech Stack)
- **Frontend**: Streamlit
- **Backend / API**: FastAPI, Python
- **AI & NLP**: Ollama, Qwen2.5 7B (또는 EXAONE), BGE-m3 임베딩, Transformers 기반 구조화
- **Database (Vector)**: ChromaDB (또는 FAISS)
- **Deployment**: 로컬 실행 및 Docker 기반(선택적)

## 🛣️ 마일스톤 (Milestones) (총 8주)
- **W 1-2 (M1)**: 요구사항 확정, 데이터 샘플셋 확보, 기술 스택 검증 (임베딩 벤치마크, Ollama 연동 PoC)
- **W 3-4 (M2)**: 민원 데이터 파이프라인(ingest) 구축, 4요소 구조화/NER 모듈 개발 및 평가(스키마 통과율 90% 달성)
- **W 5-6 (M3)**: 임베딩 및 벡터 인덱싱, 하이브리드 검색 품질 고도화, FastAPI 검색 백엔드 UI 통합 연동
- **W 7-8 (M4)**: RAG 프롬프트 구현, 출처 근거 하이라이팅 적용 기능 완성, 성능 프로파일링 및 데모/문서화 최종 발표
